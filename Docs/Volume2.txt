**Version:** 1.0

**Project:** TripKart

**Architecture:** Plugin-Based Travel Engine

---

# 1. Architecture Overview

TripKart is built using a **Plugin-Based Travel Engine**.

The application is **not** divided into Bus, Flight, Train or Hotel modules.

Instead, it is divided into:

* Core Platform
* Shared Components
* Business Engines
* Travel Plugins

This allows adding new travel services without changing the core application.

---

# 2. Architecture Principles

The application follows these principles.

### Reusability

Every screen should be reusable.

No duplicated ViewControllers.

---

### Configuration Driven

Screens should render based on configuration.

Not business type.

---

### Plugin Based

Bus, Flight, Train...

are plugins.

Not modules.

---

### Backend Ready

Replacing Dummy JSON with REST APIs should require changing only the Repository layer.

---

### Single Responsibility

Every class has one responsibility.

---

### Dependency Injection

No object creates its own dependencies.

Everything is injected.

---

### Coordinator Navigation

ViewControllers never navigate.

Only Coordinators.

# 26. Summary

TripKart is built as a reusable travel platform rather than a collection of individual booking modules.

The Core Platform owns navigation, booking, payment, tickets, analytics, storage, and business engines.

Travel plugins contribute only their unique business configurations, allowing new travel categories to be introduced with minimal effort while keeping the application clean, maintainable, and backend-ready.
